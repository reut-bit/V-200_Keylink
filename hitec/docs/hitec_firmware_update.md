# Прошивка Hitec SG50BL: обновление firmware через Pixhawk SLCAN (Windows)

## Цель

Обновить firmware сервомашинки Hitec SG50BL с **Type C (CAN 2.0A only, v1.5(2))** на **Type A/U (с поддержкой DroneCAN, v1.6(5))**. Без обновления firmware запись `REG_CAN_MODE = 2` не даёт эффекта — серво физически неспособна работать в DroneCAN.

## Необходимое оборудование

- Компьютер с Windows 10/11
- Pixhawk (в роли SLCAN-адаптера, подключён к CAN шине)
- USB-кабель Pixhawk → ПК
- Сервомашинка Hitec SG50BL, подключённая к CAN шине (24В питание)
- QGroundControl установлен и **закрыт** во время прошивки

> **Вероятность успеха без DPC-CAN: ~50%.** Если AIO UI увидит серво при подключении — прошивка пройдёт. Если нет — потребуется фирменный программатор Hitec DPC-CAN.

## Шаг 1: Скачать необходимые файлы

Перейти на страницу: [http://support.hitecrcd.net:7700/index.php/CanServo/FW_Files/en](http://support.hitecrcd.net:7700/index.php/CanServo/FW_Files/en)

Скачать:

| Файл | Назначение |
|------|-----------|
| `CAN_UPDATE_V2-2022_09_14_01.zip` | AIO UI — утилита для прошивки (или актуальную версию из раздела "AIO UI") |
| `App_SG50_CAN,UAVCAN_20_25_08_20_01_V1.6(5).zip` | Firmware SG50BL (CAN + DroneCAN, Type A/U) |
| .NET Framework 4 | Если не установлен (ссылка есть на странице Hitec) |

Распаковать архив AIO UI в удобную папку (например, `C:\Hitec_AIO`).

---

## Шаг 2: Настроить Pixhawk в режим SLCAN

**Важно:** QGroundControl должен быть **закрыт** — иначе он займёт COM-порт.

### 2.1 Открыть QGC и выставить параметры

Открыть QGroundControl, подключиться к Pixhawk, перейти в **Vehicle Setup → Parameters**:

| Параметр | Значение | Примечание |
|----------|----------|------------|
| `CAN_SLCAN_CPORT` | `1` | SLCAN на CAN1 |
| `CAN_SLCAN_TIMOUT` | `0` | Без таймаута (не отключаться) |

Нажать **Reboot Vehicle** (перезагрузить Pixhawk).

### 2.2 После перезагрузки — включить SLCAN

После того как Pixhawk снова подключился к QGC, выставить **без перезагрузки**:

| Параметр | Значение |
|----------|----------|
| `CAN_SLCAN_SERNUM` | `0` |

> Этот параметр сбрасывается автоматически при следующей загрузке. Его нужно выставлять каждый раз после перезагрузки Pixhawk, без повторного ребута.

### 2.3 Закрыть QGroundControl

**Полностью закрыть QGC** — Pixhawk теперь готов к работе как SLCAN-адаптер.

### 2.4 Найти номер COM-порта

Открыть **Диспетчер устройств** (`Win + X → Диспетчер устройств → Порты COM и LPT`).

Найти Pixhawk — обычно отображается как `COMx` (например, `COM5` или `COM7`). Запомнить номер.

## Шаг 3: Запустить AIO UI и подключиться к серво

1. Запустить `CAN_UPDATE_V2.exe` из распакованной папки
2. В поле **Port** выбрать COM-порт Pixhawk (например, `COM5`)
3. В поле **Baudrate** выбрать **1000000** (1 Мбит/с — серво уже настроена на эту скорость)
4. Нажать кнопку **Connect**

**Ожидаемый результат:** серво отобразится в списке устройств с информацией о версии прошивки (`REG_VERSION = 11052`, Product No = SG50).

> Если серво **не определилась** — AIO UI несовместим с Pixhawk SLCAN протоколом. Потребуется DPC-CAN.  
> Если определилась — продолжать.

---

## Шаг 4: Обновить firmware

1. Перейти на вкладку **FW Update** (или **Firmware Update**)
2. Нажать **Browse / Load** и выбрать файл:
   ```
   App_SG50_CAN,UAVCAN_20_25_08_20_01_V1.6(5).zip
   ```
3. Убедиться, что выбрана правильная серво в списке
4. Нажать **Update / Flash / Start**
5. **Не отключать питание серво и USB** во время прошивки
6. Дождаться завершения (обычно 1–3 минуты, на экране будет прогресс-бар)

---

## Шаг 5: Проверить результат

После завершения прошивки:

1. **Отключить питание серво на 5 секунд**, затем включить снова
2. В AIO UI нажать **Re-scan / Refresh** — серво должна снова появиться
3. Проверить версию: `REG_VERSION` должна быть отличной от `11052` (например, `16050` для v1.6(5))

---

## Шаг 6: Записать DroneCAN параметры (macOS / Linux)

После обновления firmware вернуться к Mac и запустить скрипт прошивки DroneCAN:

```bash
cd "/Users/reutov/Work/ICAR center/750/V-200_Keylink"
source .venv/bin/activate

# Сначала проверить, что серво откликается
python scripts/scan_can1_hitec.py /dev/cu.usbmodem14301 1000000

# Записать DroneCAN параметры (Node ID = 55 для дросселя)
python scripts/flash_hitec_dronecan.py /dev/cu.usbmodem14301 55
```

Убедиться что в выводе скрипта:
- `REG_CAN_MODE = 2` ✓
- `REG_CAN_BAUDRATE = 0` (1000 kbps) ✓
- `REG_ID = 55` ✓
- `REG_SAMPLE_POINT = 1` ✓

---

## Шаг 7: Проверить в dronecan_gui_tool

```bash
dronecan_gui_tool
```

Подключиться через MAVLink к Pixhawk. Нода **55** должна появиться в списке с именем в духе `org.hitec.SG50` или аналогичным.

## Возможные проблемы

| Проблема | Причина | Решение |
|----------|---------|---------|
| AIO UI не видит COM-порт | QGC не закрыт / Pixhawk не инициализирован | Закрыть QGC, повторить шаг 2 |
| Серво не определяется при Connect | AIO UI несовместим с Pixhawk SLCAN | Использовать DPC-CAN |
| Ошибка во время прошивки | Нестабильное питание / помехи | Обеспечить стабильное 24В, повторить |
| После прошивки серво не реагирует | Firmware повреждён | Повторить прошивку, держа питание |
| REG_VERSION не изменился | Прошивка для неправильной модели | Проверить Product No серво и выбрать нужный файл |

## Альтернатива: DPC-CAN

Если Pixhawk SLCAN не совместим с AIO UI, заказать **Hitec DPC-CAN** (USB-CAN программатор):
- AliExpress: поиск "Hitec DPC-CAN" или "SG Series CAN Programmer"
- Официально: [hitecrcd.com](https://hitecrcd.com)
- Цена: ~$30–60
- Подключение: напрямую к CAN шине (CAN H/L), USB → ПК
