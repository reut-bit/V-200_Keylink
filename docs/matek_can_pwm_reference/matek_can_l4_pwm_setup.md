# Инструкция по настройке Matek CAN-L4-PWM

**Плата:** Matek AP_Periph CAN-L4-PWM  
**Полётный контроллер:** Pixhawk6X Pro  
**Прошивка:** ArduPilot (ArduCopter/Heli) 4.6.3  
**Режим работы:** только PWM (DShot НЕ используется)  
**Источник:** [Using MatekL431 adapters for PWM and DShot](https://discuss.ardupilot.org/t/using-matekl431-adapters-for-pwm-and-dshot/85781)

---

## 1. Общее описание

CAN-L4-PWM — это DroneCAN-нода на базе STM32L431, которая получает команды от полётного контроллера по CAN-шине и преобразует их в PWM-сигналы для сервоприводов и других периферийных устройств.

**Характеристики платы:**
- 9 PWM-выходов (пады 1–9)
- 1 CAN-порт (JST-GH 4-pin)
- Питание: 4.5–5.5V на пад "5V"
- Пады "Vx" — для внешнего питания серво (не связаны с логикой платы)

**Ограничение таймеров:**
- Выходы 1–4 сидят на одном таймере (TIM1) → все 4 должны быть ЛИБО серво, ЛИБО ESC. Смешивать нельзя.
- Выходы 5–8 на другом таймере (TIM2).
- Выход 9 (TIM16) — отдельный, без DMA.

## 2. Подготовка платы (прошивка)

### 2.1. Установка прошивки MatekL431-DShot

Несмотря на название, эта прошивка поддерживает и PWM-режим.

1. Скачать последнюю прошивку:  
   https://firmware.ardupilot.org/AP_Periph/latest/MatekL431-DShot/

2. DroneCAN GUI Tool уже установлен на этом компьютере (`dronecan_gui_tool`).

3. Подключить плату к CAN-порту полётного контроллера.

4. Подключить компьютер к Pixhawk по Ethernet (IP компьютера: 192.168.144.10).

5. Запустить DroneCAN GUI Tool:
   ```bash
   dronecan_gui_tool
   ```
   В окне подключения:
   - **Interface:** `mavcan:0.0.0.0:14550`
   - **Bus Number:** `2` (CAN2) или `1` (CAN1)
   - **CAN bus bit rate:** `1000000`
   - Нажать **OK**
   - В главном окне нажать галочку у **Node ID: 127**

6. В списке нод найти CAN-L4-PWM → нажать **Update Firmware** → выбрать файл прошивки из:
   ```
   docs/firmware matek/AP_Periph.apj
   ```

7. Дождаться завершения прошивки, нода перезагрузится.

## 3. Настройка параметров на CAN-ноде

Параметры ноды настраиваются через **DroneCAN GUI Tool** или через **Mission Planner** (Setup → Optional Hardware → DroneCAN/UAVCAN).

### 3.1. Отключение DShot (обязательно для PWM)

| Параметр ноды | Значение | Описание |
|---|---|---|
| `OUT_BLH_MASK` | **0** | Отключить DShot на всех выходах |
| `ESC_PWM_TYPE` | **0** | Режим обычного PWM |

### 3.2. Назначение функций на выходы

Каждый выход ноды привязывается к конкретному каналу серво полётного контроллера по формуле:

> **OUTn_FUNCTION = 50 + номер SERVO на полётном контроллере**

Например, если на полётном контроллере `SERVO1` управляет циклическим шагом, и вы хотите вывести этот сигнал на выход 1 ноды:

| Параметр ноды | Значение | Что делает |
|---|---|---|
| `OUT1_FUNCTION` | **51** | Выход 1 ноды = SERVO1 ПК (50+1) |
| `OUT2_FUNCTION` | **52** | Выход 2 ноды = SERVO2 ПК (50+2) |
| `OUT3_FUNCTION` | **53** | Выход 3 ноды = SERVO3 ПК (50+3) |
| `OUT4_FUNCTION` | **54** | Выход 4 ноды = SERVO4 ПК (50+4) |
| `OUT5_FUNCTION` | **55** | Выход 5 ноды = SERVO5 ПК (50+5) |
| ... | ... | ... |

**Неиспользуемые выходы ОБЯЗАТЕЛЬНО выставить в 0:**

| Параметр ноды | Значение |
|---|---|
| `OUTn_FUNCTION` | **0** (для всех неиспользуемых) |

> **Важно:** не оставляйте неиспользуемые выходы со значениями по умолчанию (33+). Это значения для ESC, и они будут конфликтовать с PWM-режимом.

### 3.3. Диапазоны PWM на ноде

Рекомендуется выставить стандартные диапазоны на ноде, чтобы управление min/max/trim шло с полётного контроллера:

| Параметр ноды | Значение |
|---|---|
| `OUTn_MIN` | **1000** |
| `OUTn_TRIM` | **1500** |
| `OUTn_MAX` | **2000** |

Это означает, что значение PWM в логах полётного контроллера будет точно соответствовать тому, что выдаёт нода.

### 3.4. Сводная таблица параметров ноды (пример для 4 серво)

| Параметр | Значение | Комментарий |
|---|---|---|
| `ESC_PWM_TYPE` | 0 | Обычный PWM |
| `OUT_BLH_MASK` | 0 | DShot выключен |
| `OUT1_FUNCTION` | 51 | → SERVO1 |
| `OUT2_FUNCTION` | 52 | → SERVO2 |
| `OUT3_FUNCTION` | 53 | → SERVO3 |
| `OUT4_FUNCTION` | 54 | → SERVO4 |
| `OUT5_FUNCTION` | 0 | Не используется |
| `OUT6_FUNCTION` | 0 | Не используется |
| `OUT7_FUNCTION` | 0 | Не используется |
| `OUT8_FUNCTION` | 0 | Не используется |
| `OUT9_FUNCTION` | 0 | Не используется |
| `OUT1_MIN` | 1000 | |
| `OUT1_TRIM` | 1500 | |
| `OUT1_MAX` | 2000 | |
| *(аналогично для OUT2–OUT4)* | | |

## 4. Настройка параметров на полётном контроллере

### 4.1. Включение CAN-шины

CAN-порт и DroneCAN-драйвер должны быть включены (у нас уже настроено):

| Параметр ПК | Значение | Описание |
|---|---|---|
| `CAN_P1_DRIVER` | 1 | CAN1 → драйвер 1 |
| `CAN_P1_BITRATE` | 1000000 | 1 Мбит/с |
| `CAN_D1_PROTOCOL` | 1 | DroneCAN |

Если нода подключена к CAN2:

| Параметр ПК | Значение |
|---|---|
| `CAN_P2_DRIVER` | 2 |
| `CAN_P2_BITRATE` | 1000000 |
| `CAN_D2_PROTOCOL` | 1 |

### 4.2. Включение передачи серво-команд по CAN

| Параметр ПК | Значение | Описание |
|---|---|---|
| `CAN_D1_UC_SRV_BM` | **битовая маска** | Какие SERVO передавать по CAN |
| `CAN_D1_UC_SRV_RT` | **50** | Частота обновления серво (Гц) |

**Битовая маска `CAN_D1_UC_SRV_BM`:**

Каждый бит соответствует номеру SERVO (считая с 1):

| SERVO каналы | Битовая маска | Десятичное |
|---|---|---|
| SERVO1 | 0b0000 0001 | 1 |
| SERVO1–2 | 0b0000 0011 | 3 |
| SERVO1–4 | 0b0000 1111 | 15 |
| SERVO1–8 | 0b1111 1111 | 255 |
| SERVO5–8 | 0b1111 0000 | 240 |

Например, для передачи SERVO1–SERVO4 по CAN:
- `CAN_D1_UC_SRV_BM` = **15**

### 4.3. Отключение Safety Switch (если нет физической кнопки)

Pixhawk6X Pro может не иметь кнопки Safety. Без этого CAN-нода не будет выдавать PWM:

| Параметр ПК | Значение | Описание |
|---|---|---|
| `BRD_SAFETY_DEFLT` | **0** | Safety выключен по умолчанию |

Или через Mission Planner: `Ctrl+F` → `Toggle Safety Switch`.

## 5. Подключение (аппаратная часть)

### 5.1. Схема подключения

```
Pixhawk6X Pro                    CAN-L4-PWM                     Сервоприводы
┌──────────┐    JST-GH 4pin     ┌──────────┐    PWM             ┌─────────┐
│  CAN1    ├───────────────────►│  CAN     │                    │         │
│  или     │  CAN_H, CAN_L,    │          │  Pad 1 ──────────►│ Серво 1 │
│  CAN2    │  GND, +5V         │          │  Pad 2 ──────────►│ Серво 2 │
└──────────┘                    │          │  Pad 3 ──────────►│ Серво 3 │
                                │          │  Pad 4 ──────────►│ Серво 4 │
                                │    5V ◄──┤  (питание логики) │         │
                                │    Vx ◄──┤  (питание серво   │         │
                                │          │   от внешнего BEC) └─────────┘
                                └──────────┘
```

### 5.2. Питание

- **Логика платы (пад "5V"):** 4.5–5.5V — питается от CAN-разъёма (Pixhawk подаёт 5V по CAN).
- **Сервоприводы (пады "Vx"):** подключить внешний BEC. Пады "Vx" **не связаны** с "5V" на плате. Серво питаются отдельно!

> **Внимание:** НЕ подавайте питание серво через пад "5V" — это перегрузит линию питания CAN.

### 5.3. Терминирование CAN-шины

На конце CAN-шины должен стоять терминирующий резистор **120 Ом** между CAN_H и CAN_L. Если CAN-L4-PWM — последнее устройство на шине, впаяйте резистор или используйте внешний терминатор.

## 6. Проверка работоспособности

### 6.1. Проверка через DroneCAN GUI Tool

1. Закрыть QGroundControl (занимает порт 14550).
2. Запустить `dronecan_gui_tool`.
3. Подключиться:
   - **Interface:** `mavcan:0.0.0.0:14550`
   - **Bus Number:** `2` (CAN2) или `1` (CAN1)
   - **CAN bus bit rate:** `1000000`
4. Нажать галочку у **Node ID: 127** (без этого discovery не работает).
5. В списке нод должна появиться CAN-L4-PWM с присвоенным Node ID.
6. Открыть **Bus Monitor** (Tools → Bus Monitor).
7. Должны быть видны пакеты `actuator.ArrayCommand` с данными серво.

### 6.2. Проверка через скрипт can_scan.py

```bash
# QGC должен быть закрыт
cd mavlink_tools
./can_scan.sh eth
```

Скрипт покажет все обнаруженные CAN-устройства через параметры автопилота (DEVID).

### 6.3. Тест сервоприводов

В Mission Planner или QGC:
- Перейти в Motor/Servo Test
- Двигать каналы серво
- Сервоприводы на CAN-ноде должны отрабатывать

## 7. Диагностика проблем

| Проблема | Причина | Решение |
|---|---|---|
| GUI Tool: нод не видно | Node ID не задан | Нажать галочку у Node ID 127 в главном окне |
| GUI Tool: нод не видно | Неверный Bus Number | Bus Number: 1=CAN1, 2=CAN2 (в окне настройки) |
| GUI Tool: не подключается | QGC занимает порт | Закрыть QGroundControl перед запуском |
| GUI Tool: ошибка подключения | Неверный формат | Interface: `mavcan:0.0.0.0:14550` (без `udpin:`) |
| Нода не появляется | CAN-порт не включён | Проверить `CAN_Px_DRIVER` ≠ 0, перезагрузить |
| Нода есть, но серво не двигаются | Safety не отключён | `BRD_SAFETY_DEFLT=0` или Toggle Safety |
| Нода есть, но серво не двигаются | `CAN_D1_UC_SRV_BM` = 0 | Выставить битовую маску нужных каналов |
| Серво дёргается / не тот канал | Неверный `OUTn_FUNCTION` | Пересчитать: 50 + номер SERVO на ПК |
| PWM выходит за пределы | Разные min/max на ноде и ПК | На ноде: 1000/1500/2000, управлять через ПК |
| Нода не прошивается | Старая прошивка | Попробовать через STM32 DFU (замкнуть BOOT0) |

## 9. Управление БАНО от тумблера пульта (через CAN-L4-PWM)

### Принцип

```
Тумблер пульта → RC-канал (напр. RC6) → Pixhawk SERVO6 (RCPassThru) → CAN → OUT6 ноды → БАНО (PWM)
```

Логика сигнала: тумблер вниз = 1000 мкс = ВЫКЛ, тумблер вверх = 2000 мкс = ВКЛ.

---

### 9.1. Назначение тумблера на RC-канал (пульт + QGC)

1. На пульте назначить тумблер на вспомогательный канал (например, **CH6 → RC6**).
   - Позиция «выкл» = 1000 мкс
   - Позиция «вкл» = 2000 мкс

2. В QGC: **Radio Setup** → убедиться, что RC6 движется при переключении тумблера.

---

### 9.2. Параметры Pixhawk в QGC

| Параметр ПК | Значение | Описание |
|---|---|---|
| `SERVO6_FUNCTION` | **1** | RCPassThru — напрямую передаёт сигнал RC6 |
| `SERVO6_MIN` | **1000** | |
| `SERVO6_TRIM` | **1000** | По умолчанию ВЫКЛ |
| `SERVO6_MAX` | **2000** | |
| `SERVO6_REVERSED` | **0** | Если логика перевёрнута — поставить 1 |
| `CAN_D1_UC_SRV_BM` | включить бит 6 | Добавить SERVO6 в битовую маску CAN-передачи |

**Как добавить SERVO6 в `CAN_D1_UC_SRV_BM`:**

Если, например, уже передаются SERVO1–4 (маска = 15 = `0b00001111`), добавить бит 6:

| Добавляемый канал | Добавочное значение | Пример итога |
|---|---|---|
| + SERVO5 | +16 | 31 |
| + SERVO6 | +32 | 63 |
| + SERVO7 | +64 | 127 |
| + SERVO8 | +128 | 255 |

> Т.е. если до этого маска была `15`, для добавления SERVO6 ставим `15 + 32 = 47`.

---

### 9.3. Параметры CAN-ноды (через DroneCAN GUI Tool)

| Параметр ноды | Значение | Описание |
|---|---|---|
| `OUT6_FUNCTION` | **56** | 50 + 6 = SERVO6 полётного контроллера |
| `OUT6_MIN` | **1000** | |
| `OUT6_TRIM` | **1000** | |
| `OUT6_MAX` | **2000** | |

---

### 9.4. Проверка

1. Включить пульт, перейти в QGC → **Analyze → Mavlink Inspector**, найти `SERVO_OUTPUT_RAW`.
2. При переключении тумблера `servo6_raw` должен меняться между 1000 и 2000.
3. На выходе `Pad 6` CAN-ноды мультиметром: ~+3.3В = ВКЛ, ~0В = ВЫКЛ (или осциллограф: PWM 1000/2000 мкс).

---

### 9.5. Если тумблер не на RC6

Если переключатель назначен на другой RC-канал (например, RC8), схема аналогична — просто используйте `SERVO8_FUNCTION = 1` и `OUT8_FUNCTION = 58`. RCPassThru всегда работает по правилу: **SERVOn_FUNCTION = 1** передаёт **RCn** того же номера.

---

## 8. Несколько нод на одной шине

Можно подключить несколько CAN-L4-PWM к одной CAN-шине. Каждая нода получит уникальный Node ID автоматически. Нужно только правильно назначить `OUTn_FUNCTION` на каждой ноде, чтобы они слушали разные каналы серво.

Терминирующий резистор (120 Ом) ставится только на **двух концах** шины (первое и последнее устройство).

## 10. Полная настройка PWM-реле через CAN-L4-PWM (с нуля)

Подключаем PWM-реле к **выходу 5 (Pad 5)** модуля Matek CAN-L4-PWM.
Управление — от тумблера на пульте.

Всё оборудование только что из коробки: Pixhawk6X Pro, Matek CAN-L4-PWM, PWM-реле.

### 10.1. Что потребуется

| Компонент | Описание |
|---|---|
| **Pixhawk6X Pro** | Полётный контроллер с прошивкой ArduPilot 4.6+ |
| **Matek CAN-L4-PWM** | DroneCAN → PWM адаптер (9 выходов) |
| **PWM-реле** | Модуль реле, управляемый PWM-сигналом (>1500 мкс = ВКЛ, <1500 мкс = ВЫКЛ) |
| **Кабель CAN** | JST-GH 4-pin (идёт в комплекте с Matek) |
| **БП / BEC** | Внешний источник питания для нагрузки реле (на пад Vx) |
| **ПК** | С установленным QGroundControl и `dronecan_gui_tool` |
| **Ethernet-кабель** | Для подключения ПК к Pixhawk |

### 10.2. Принцип работы

```
Тумблер пульта → RC10 → Pixhawk (SERVO10, RCPassThru) → CAN1 → Matek OUT5 → PWM → Реле
```

- Тумблер вниз = PWM 1000 мкс = реле ВЫКЛ
- Тумблер вверх = PWM 2000 мкс = реле ВКЛ

Используем канал **RC10 → SERVO10** на полётнике и выход **OUT5** на ноде.
Номер RC-канала и номер выхода на ноде — независимые вещи, связь между ними задаётся параметром `OUT5_FUNCTION`.

---

### 10.3. Физическое подключение

```
                                   Matek CAN-L4-PWM
Pixhawk6X Pro                     ┌─────────────────┐
┌──────────┐    JST-GH 4pin      │                 │
│          ├─────────────────────►│ CAN             │          PWM-реле
│  CAN1    │  CAN_H, CAN_L,      │                 │         ┌────────┐
│          │  GND, +5V            │  Pad 5 (signal)─┼────────►│ SIG    │
└──────────┘                      │  Pad 5 (GND)  ──┼────────►│ GND    │
                                  │  Vx ◄───────────┤         │ V+ ◄───┤── БП / BEC
                                  │                 │         └────────┘
                                  │  5V ◄── (от CAN)│
                                  └─────────────────┘
                                        ▲
                                        │ 120 Ом (если последнее устройство на шине)
```

**Питание:**
- **Логика платы Matek** (пад `5V`): получает 5V от Pixhawk через CAN-разъём.
- **Пад `Vx`**: для питания нагрузки реле от внешнего BEC. **НЕ связан** с 5V логики.
- **PWM-реле**: сигнальный провод (SIG) → к паду 5 (signal), GND → к паду 5 (GND).

**CAN-терминирование:**
Если Matek — последнее устройство на CAN-шине, впаять резистор **120 Ом** между CAN_H и CAN_L на плате (или использовать внешний терминатор).

5. Запустить DroneCAN GUI Tool:
   ```bash
   dronecan_gui_tool
   ```
   Настройки подключения:
   - **Interface:** `mavcan:0.0.0.0:14550`
   - **Bus Number:** `1` (CAN1)
   - **CAN bus bit rate:** `1000000`
   - Нажать **OK**
   - Поставить галочку у **Node ID: 127**

6. В списке нод найти CAN-L4-PWM → **Update Firmware** → выбрать скачанный файл `.apj`.

7. Дождаться завершения, нода перезагрузится.

| `OUT4_FUNCTION` | **0** |

### 10.6. Параметры Pixhawk (QGroundControl)

#### CAN-шина

Включение CAN1 и DroneCAN (если ещё не настроено):

| Проблема | Причина | Решение |
|---|---|---|

## 11. Управление MOSFET через GPIO (relay) на выходе 7 CAN-L4-PWM

Мосфет управляется цифровым сигналом (0V = ВЫКЛ, 3.3V = ВКЛ), а не PWM.
ArduPilot поддерживает **удалённые реле через DroneCAN** — полётник отправляет команду hardpoint по CAN, нода переключает физический пин.

Прошивка MatekL431-DShot поддерживает relay (`AP_PERIPH_RELAY_ENABLED`, `AP_RELAY_ENABLED` подтверждено в [features.txt](https://firmware.ardupilot.org/AP_Periph/latest/MatekL431-DShot/features.txt)).

### 11.1. Принцип работы

```
Тумблер пульта → RC9 → Pixhawk (RC9_OPTION=28, Relay1) → CAN hardpoint → Matek GPIO (Pad 7) → MOSFET
```

В отличие от PWM-реле (раздел 10), здесь выход работает как **чистый GPIO**: 0V или 3.3V, без ШИМ-модуляции. Это правильный режим для управления мосфетом.

### 11.2. Подключение

```
Matek CAN-L4-PWM                  MOSFET
┌─────────────────┐               ┌──────────┐
│  Pad 7 (signal)─┼──────────────►│ Gate     │
│  Pad 7 (GND)  ──┼──────────────►│ Source   │
│                 │               │ Drain ───┼──► Нагрузка
│  Vx ◄───────────┤               └──────────┘
└─────────────────┘
```

- **Gate** мосфета — к сигнальному пину Pad 7
- **Source** — к GND Pad 7
- **Drain** — к нагрузке
- Питание нагрузки через Vx / внешний БП

> Выход GPIO даёт 3.3V / ~8 мА. Для мощных мосфетов с высоким порогом (>3.3V) может потребоваться gate driver. Для logic-level мосфетов (порог 1.5–2.5V) 3.3V достаточно.

### 11.3. Параметры CAN-ноды (DroneCAN GUI Tool)

| Параметр ноды | Значение | Описание |
|---|---|---|
| `OUT7_FUNCTION` | **-1** | Режим GPIO (не серво/ESC) |
| `RELAY1_FUNCTION` | **10** | DroneCAN remote relay |
| `RELAY1_PIN` | **56** | GPIO-пин выхода 7 (нумерация: OUT1=50, OUT2=51, ..., OUT7=56) |

Остальные выходы, если не используются, оставить `OUTn_FUNCTION = 0`.

**Нумерация GPIO-пинов на Matek CAN-L4-PWM:**

| Выход | GPIO-пин |
|---|---|
| OUT1 | 50 |
| OUT2 | 51 |
| OUT3 | 52 |
| OUT4 | 53 |
| OUT5 | 54 |
| OUT6 | 55 |
| OUT7 | **56** |
| OUT8 | 57 |
| OUT9 | 58 |

### 11.8. Отличия от PWM-реле (раздел 10)

| | PWM-реле (раздел 10) | GPIO/MOSFET (раздел 11) |
|---|---|---|
| Сигнал на выходе | ШИМ (1000–2000 мкс) | Цифровой (0V / 3.3V) |
| Подходит для | PWM-реле, серво | MOSFET, твердотельные реле |
| Путь команды | SERVO → CAN → OUTn PWM | RELAY → CAN hardpoint → GPIO |
| Параметр на ноде | `OUTn_FUNCTION = 50+x` | `OUTn_FUNCTION = -1`, `RELAY_PIN` |
| Параметр на ПК | `SERVOn_FUNCTION`, `SRV_BM` | `RELAY1_PIN = 1000`, `RCx_OPTION = 28` |
| Управление из миссии | `DO_SET_SERVO` | `DO_SET_RELAY` |

### 11.9. Диагностика

| Проблема | Причина | Решение |
|---|---|---|
| Пин не переключается | `OUT7_FUNCTION` ≠ -1 | Выставить -1, перезагрузить ноду |
| Пин не переключается | `RELAY1_PIN` неверный | Проверить: OUT7 = пин 56 |
| Relay не работает | `RELAY1_FUNCTION` на ноде ≠ 10 | Поставить 10 (DroneCAN relay) |
| Relay не работает | `RELAY1_PIN` на ПК ≠ 1000 | 1000 = виртуальный CAN-relay |
| Тумблер не управляет | `RC9_OPTION` ≠ 28 | 28 = Relay1 On/Off |
| Мосфет не открывается | Порог Gate > 3.3V | Использовать logic-level мосфет или gate driver |
| Прошивка не поддерживает relay | Старая прошивка | Обновить MatekL431-DShot до последней версии |
