# Настройка Hitec SG50BL-CAN в режим DroneCAN

**Сервопривод:** Hitec SG50BL-CAN-24V (Part #34501)
**Протокол по умолчанию:** CAN 2.0A (250 kbps)
**Целевой протокол:** DroneCAN / UAVCAN v0 (1000 kbps)
**Полётный контроллер:** Pixhawk6X Pro, ArduPilot 4.6.3
**CAN-шина:** CAN1 (`CAN_P1_DRIVER=1`, `CAN_D1_PROTOCOL=1`, `CAN_P1_BITRATE=1000000`)

---

## 1. Зачем переключать режим

Hitec SG50BL по умолчанию работает в проприетарном протоколе CAN 2.0A.
Фреймы этого протокола **несовместимы** с DroneCAN и вызывают ошибки на шине:

- `TransferError: Toggle bit value incorrect` — нарушение порядка фреймов
- `TransferError: CRC mismatch` — повреждение данных

Пока сервоприводы в режиме CAN 2.0A, они **невидимы** для ArduPilot и DroneCAN GUI Tool,
а их трафик **ломает** связь с другими DroneCAN-устройствами на той же шине.

---

## 2. Необходимое оборудование

| Компонент                       | Назначение                               |
| ------------------------------- | ---------------------------------------- |
| SG Series DroneCAN/CAN Dongle  | CAN-адаптер для подключения серво к ПК   |
| USB-TTL переходник              | Связь донгла с ПК (115200 bps, 8N1)     |
| Блок питания 24В                | Питание серво (диапазон 18–32В)          |
| ПК с Windows                    | Hitec Configuration App — только Windows |

### Схема подключения

```text
[PC] ──USB──▶ [USB-TTL] ──TTL 115200──▶ [DPC-CAN Dongle] ──CAN──▶ [Servo SG50BL]
                                                                         │
                                                                    [БП 24В]
```

> **Важно:** подключать строго **по одному серво**.
> У всех заводской ID = 0 — при подключении нескольких одновременно будет конфликт адресов.

## 4. Регистры для настройки

### 4.1. Параметры, одинаковые для всех 5 серво

| Регистр              | Адрес | Заводское     | Новое              | Описание                                       |
| -------------------- | ----- | ------------- | ------------------ | ---------------------------------------------- |
| `REG_CAN_BAUDRATE`   | 0x38  | 5 (250 kbps)  | **0** (1000 kbps)  | Должно совпадать с `CAN_P1_BITRATE`            |
| `REG_CAN_MODE`       | 0x6A  | 0 (CAN 2.0A)  | **2** (DroneCAN)   | Переключение протокола                         |
| `REG_CAN_BUS_ID_H`   | 0x3C  | 0             | **0**              | Для DroneCAN всегда 0                          |
| `REG_CAN_BUS_ID_L`   | 0x3E  | 0             | **128**            | 128 = игнорировать Src Node ID в ArrayCommand  |
| `REG_SAMPLE_POINT`   | 0x40  | 0 (50%)       | **1** (87.5%)      | Рекомендуется для DroneCAN                     |
| `REG_RUN_MODE`       | 0x44  | 1             | **1** (Servo)      | Режим работы — Servo Mode                      |

### 4.2. Уникальный Node ID для каждого серво

| Серво   | `REG_ID` (0x32) | Назначение |
| ------- | --------------- | ---------- |
| Серво 1 | **10**          | —          |
| Серво 2 | **11**          | —          |
| Серво 3 | **12**          | —          |
| Серво 4 | **13**          | —          |
| Серво 5 | **14**          | —          |

> Node ID должны быть уникальны на всей CAN-шине (1–127).
> Диапазон 10–14 выбран, чтобы не конфликтовать с автопилотом (Node 10 по `CAN_D1_UC_NODE`)
> и другими DroneCAN-нодами на CAN2.

**Если `CAN_D1_UC_NODE=10` конфликтует с серво 1** — сдвинуть диапазон на 20–24
или изменить `CAN_D1_UC_NODE` в ArduPilot.

## 5. Пошаговая процедура (для каждого серво)

### Шаг 1. Физическое подключение

1. Подключить USB-TTL к ПК
2. Подключить TTL-выход к DPC-CAN Dongle
3. Подключить CAN-выход донгла к серво
4. Подать питание 24В на серво
5. Убедиться, что подключено **только одно серво**

### Шаг 2. Подключение в Configuration App

1. Запустить Hitec CAN Configuration App
2. Выбрать COM-порт USB-TTL адаптера
3. Скорость: **115200 bps**
4. Нажать Connect
5. Считать текущие параметры серво

### Шаг 3. Изменение параметров

Записать следующие регистры:

```text
REG_CAN_BAUDRATE (0x38) = 0      # 1000 kbps
REG_CAN_MODE     (0x6A) = 2      # DroneCAN
REG_CAN_BUS_ID_H (0x3C) = 0      # всегда 0 для DroneCAN
REG_CAN_BUS_ID_L (0x3E) = 128    # игнорировать Src Node ID
REG_SAMPLE_POINT (0x40) = 1      # 87.5%
REG_ID           (0x32) = XX     # уникальный Node ID (20/21/22/23/24)
```

### Шаг 4. Сохранение и перезагрузка

1. **Остановить серво** (если вращается)
2. **Записать** `0xFFFF` в `REG_CONFIG_SAVE` (0x70) — сохранение в NVRAM
3. **Подождать** 1 секунду
4. **Перезагрузить** серво — выключить и включить питание
5. **Проверить** — повторно считать параметры, убедиться что значения сохранились

### Шаг 5. Повторить

Отключить серво, подключить следующее. Повторить шаги 1–4 с другим Node ID.

## 6. Проверка после прошивки всех 5 серво

### 6.1. DroneCAN GUI Tool

Подключить все серво к CAN1 и запустить DroneCAN GUI Tool:

```bash
dronecan_gui_tool
```

- **Interface:** `mavcan:0.0.0.0:14550`
- **Bus Number:** 1 (CAN1)
- **Node ID:** 127 → нажать Apply

В таблице нод должны появиться 5 серво с ID 10–14 и типом `Actuator`.

### 6.2. Скрипт can_scan.py

```bash
cd mavlink_tools
./can_scan.sh
```

Серво будут видны в параметрах, если ArduPilot их обнаружит (DEVID).

### 6.3. QGroundControl

В QGC → Parameters найти `SERVOx_FUNCTION` для назначения функций каждому каналу
DroneCAN-сервопривода.

## 7. Настройка ArduPilot для DroneCAN-серво

После того как серво видны на шине, в ArduPilot нужно настроить:

| Параметр            | Значение       | Описание                                      |
| ------------------- | -------------- | --------------------------------------------- |
| `CAN_D1_UC_SRV_BM` | битовая маска  | Включение каналов серво через DroneCAN        |
| `SERVOx_FUNCTION`   | по назначению  | Функция каждого канала (руль, элерон и т.д.)  |

Битовая маска `CAN_D1_UC_SRV_BM` — каждый бит включает канал:

- Бит 0 = Servo 1, Бит 1 = Servo 2, ... Бит 4 = Servo 5
- Для 5 серво (каналы 1–5): `0x1F` = **31**

## 8. Устранение неполадок

### Серво не обнаруживается в Configuration App

- Проверить COM-порт и скорость (115200)
- Проверить питание серво (18–32В)
- Убедиться, что подключено только одно серво
- Попробовать другой USB-TTL адаптер

### Серво не появляется в DroneCAN GUI Tool после прошивки

- Убедиться, что `REG_CAN_MODE=2` сохранён и серво перезагружено
- Проверить `REG_CAN_BAUDRATE=0` (1000 kbps)
- Проверить, что Node ID уникален и не конфликтует с другими устройствами
- Убедиться, что в GUI Tool выбран Bus Number = 1 (CAN1) и нажат Apply на Node ID 127

### CRC mismatch / Toggle bit errors в DroneCAN GUI Tool

Эти ошибки появляются, если на шине есть устройства в режиме CAN 2.0A.
Все серво на шине должны быть переведены в DroneCAN или **физически отключены**
до завершения перенастройки.

### F/W Type = C — невозможно переключить в DroneCAN

Если прошивка серво типа «C», режим DroneCAN недоступен.
Необходимо обновить прошивку до типа «A» через
[Hitec Wiki](http://support.hitecrcd.net:7700/index.php/CanServo/FW_Files/en).
